SPECIFIC GAPS IDENTIFIED
================================================================================

1. DEPTH CONFIDENCE (High Impact, 2 Hours)
   Reality: ARKit provides confidenceMap
   Missing: ARVOS doesn't send it
   Impact: Can't do outlier rejection in SLAM
   Fix: Add confidenceData: Data to DepthFrameMetadata

2. IMU CALIBRATION (High Impact, 1 Hour)
   Reality: Devices have known accelerometer/gyro biases
   Missing: ARVOS sends raw, biased data
   Impact: Sensor fusion algorithms inaccurate
   Fix: Send accelerometerBias, gyroscopeBias per sample

3. BATCH EXPORT (High Impact, 4 Hours)
   Reality: Researchers have 50+ sessions to process
   Missing: SDK is per-session only
   Impact: Everyone writes their own export script
   Fix: CLI tool: arvos batch-export /sessions --format csv

4. FRAME DROP DETECTION (Medium Impact, 3 Hours)
   Reality: WiFi hiccups cause silent frame loss
   Missing: No sequence number tracking
   Impact: Timestamp misalignment ruins data
   Fix: Add sequence numbers, detect gaps

5. CLOCK SYNC TRACKING (Medium Impact, 2 Hours)
   Reality: Multi-iPhone setups need sync
   Missing: No clock offset visibility
   Impact: Can't merge multiple iPhones
   Fix: Report clock offset, drift rate per frame

WHAT MAKES A TOOL "PROFESSIONAL"
================================================================================

PROFESSIONAL:
- One clear path to get data
- Tells you when something is wrong
- Reliable, reproducible
- Extensible (easy to add custom logging)
- Standard formats
- Zero decorative elements
- Can be scripted/automated

BLOATED:
- 4 presets when researchers use 1
- 10+ settings when they change 2
- Beautiful graphs nobody looks at
- Sessions export/import unneeded
- "Professional aesthetic" (visual design doesn't matter)

ARVOS's Status: 70% professional, 30% bloated

THE BOTTOM LINE
================================================================================

ARVOS is already great. The remaining work isn't "more features."

It's:
1. REMOVE bloat (presets, panels, graphs)
2. ADD metadata (confidence, calibration, uncertainty)
3. ENABLE automation (batch tools, validation)
4. FIX reliability (frame drops, clock sync)

Researchers using this don't want a beautiful app.
They want:
- My data intact
- My timestamps accurate
- My format standard
- My workflow automated
- Set it up in 30 seconds
- Walk away

Right now ARVOS does all that. Just has 20% unnecessary cruft.

The path to "professional" isn't more features.
It's less UI and more metadata.
